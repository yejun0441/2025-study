git에서 쓰는 용어

자주 쓰는 용어
    1. repository (저장소)
        말 그대로 저장소 , 로컬 저장소 & 원격 저장소 두 종류가 있다.

    2. commit (커밋)
        파일 또는 디랙토리의 변경 사항을 로컬 저장소에 기록함.
        커밋은 변경내용, 작성자, 날짜 같은 정보와 함께 고유한 ID를 가진다.

    3. Stage(스테이징)
        커밋 전에 이번에 저장할 파일만 선택하는 과정

    4. Branch(브랜치)
        작업을 분리하기 위한 수단(개발, 배포 등등 분류)
        주로 main 브랜치가 주 브랜치로 사용됨

    5. Merge(병합)
        두 브랜치의 변경사항을 하나로 합치는 과정이다.

    6. Conflict(충돌)
        두 브랜치를 합칠때 같은 파일의 같은 부분이 다르게 수정되어 Git이 병합할 수 없는 상태

GitHub(원격) 관련
    1. Remote(원격)
        GitHub에 있는 내 저장소를 가리키는 별명이다.
    
    2. Push(푸시)
        로컬 저장소의 변경 사항을 GitHub에 업로드 하는 과정
    
    3. Pull(풀)
        GitHub에 있는 것을 내 로컬 저장소로 받아온다.
    
    4. Clone(클론)
        github(원격 저장소)의 복사본을 내 PC에(로컬 저장소) 생성한다.

파일 상태 용어
    1. Tracked / Untracked
        Tracked : Git이 관리중인 파일
        Untracked : 새로 만들었는데 아직은 Git이 모르는 파일
    2. .gitignore
        디렉토리 추적 대상 제외하로록 지정하는 설정파일
